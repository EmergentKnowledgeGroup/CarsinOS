# CarsinOS Mission Control v0.1.0-beta

## Status

**Release candidate; not yet publicly released.** This document does not claim
that a GitHub Release, hosted download, live workflow result, or repository
setting has been completed.

## Windows x64 package

- Artifact: `CarsinOS-Mission-Control-v0.1.0-beta-windows-x64.msi`.
- Integrity: SHA-256 verified using the matching `SHA256SUMS.txt` and
  `release-manifest.json`.
- Signing: unsigned beta. Normal MSI administrator/UAC approval and Windows
  publisher/reputation warnings are expected.
- Runtime: Mission Control bundles a loopback-only gateway sidecar.
- State: `%LOCALAPPDATA%\io.carsinos.missioncontrol\state` is user-owned and
  remains after uninstall.

## Included operational contract

- Non-secret state backup, SHA-256 manifest verification, and restore are
  tested. Secrets, logs, locks, temporary files, and build caches are excluded.
- Restored instances require gateway/provider/channel credential re-entry.
- There is no auto-updater.
- There is no remote gateway, public hosting, proxy, tunnel, or port-forwarding
  support.
- The seven-day channel soak is explicitly excluded from this desktop-beta RC;
  it must not be represented as completed release evidence.

## Audit posture

Root Wayland/`quick-xml` advisories are temporarily ignored only on the
build-only dependency path. The nested Mission Control Tauri lockfile is
upgraded/audited separately. Informational audit debt is classified rather than
presented as a clean audit.

Current release classification after lockfile-only safe upgrades:

- Root `async-std` is an unmaintained, test-only transitive dependency of
  `httpmock`; it is not shipped in the Windows package.
- Root `ttf-parser` is an unmaintained transitive dependency of the legacy egui
  GUI; the Windows beta packages the Tauri Mission Control application instead.
- Root `paste` is retained transitively in the cross-platform lockfile but is
  not selected in the Windows dependency graph.
- Nested GTK/GLib warnings are Linux-only Tauri/WebKit dependencies and are not
  linked into the Windows x64 artifact. The remaining `rand 0.7` warning is a
  Tauri code-generation build dependency, not a runtime random-number path.
- Patched lockfile updates remove the actionable `memmap2`, `rand 0.8/0.9`,
  and nested `anyhow` warnings. No unignored RustSec vulnerability remains in
  either lockfile.

These classifications are target/reachability decisions, not permanent
waivers. Recheck both lockfiles for every beta build and remove transitive debt
when upstream dependency constraints permit it.
